Hermes — Deep Dive

by Nous Research

What is Hermes?

In plain English: Hermes is an AI assistant that takes actions on your computer and talks to you on messaging apps. Send it a message on Telegram, Discord, or the command line — it thinks about what to do, uses tools (terminal, browser, file editor), and responds. It's not just a chatbot; it's an agent that acts.

The problem it solves: LLMs can generate text but can't browse the web, run code, or remember past conversations. Hermes wraps an LLM in an "agent loop" — giving it tools, memory, and multi-step autonomy.

Who made it: Nous Research (open-source AI research org). MIT-licensed, Python, provider-agnostic — works with Anthropic, OpenAI, OpenRouter, and 10+ other backends.

What Each Box Does

  1. User Message: Raw text, voice memo, or image from any platform.
  2. Gateway / CLI Router: Entry point. The Gateway is a long-running process connecting all messaging platforms. CLI handles terminal input directly.
  3. Session Resolver: Finds/creates a conversation session per user in SQLite. Tracks history, usage, costs. Resets after idle timeout (default: 24h).
  4. Prompt Builder: Assembles the system prompt — hidden instructions the LLM reads before your message. Combines: identity (SOUL.md), memory, matched skills, context files, platform hints, and tool definitions (JSON schemas).
  5. Preflight Compression: If history already exceeds context window, compress before starting the loop.
  6. Agent Loop: Core cycle — call LLM → check for tool calls → execute → feed results back → repeat. Max 90 iterations.
  7. LLM API Call: Streaming request to provider. Tokens arrive incrementally, enabling health monitoring (90s stale detection).
  8. Tool Executor: Runs requested tools. Read-only tools run in parallel (8 workers max); writes run sequentially.
  9. Post-Loop: Agent nudged to update memory and optionally save complex workflows as skills.
  10. Save & Deliver: Conversation persisted to SQLite with FTS5 indexing. Response delivered to user's platform.

The Agent Loop — How It Actually Works

Walkthrough: "What's the weather in Berlin?"

1

You type "what's the weather in Berlin?" on Telegram.

2

The Telegram adapter receives this, resolves your session, and creates a fresh AIAgent instance with your conversation history.

3

The Prompt Builder assembles the system prompt (identity, memory, skills list, tool definitions) and appends your message to the conversation history.

4

The agent sends everything to the LLM. The LLM sees the tool definitions and decides: "I need to search the web for Berlin weather." It responds with a tool call — a structured JSON object like {"name": "web_search", "arguments": {"query": "weather Berlin today"}}.

5

The Tool Executor runs web_search, gets results, and appends them to the conversation as a "tool result" message.

6

The loop continues — the LLM now sees the search results and generates a plain text response: "It's 12°C and cloudy in Berlin today." No more tool calls, so the loop breaks.

7

The response is delivered back to Telegram. Session saved.

Key Concepts

What is a Tool Call / Function Calling?

LLMs generate text but can't run commands or browse websites. Function calling is a protocol where the LLM outputs a structured JSON request (e.g., {"name":"web_search","arguments":{"query":"..."}}) instead of text. The host executes it, returns results, and the LLM continues. Hermes registers 35+ tools with JSON schemas telling the LLM what each accepts.

Parallel Tool Execution

When the LLM requests multiple tools at once, read-only tools (read_file, web_search, vision_analyze) run simultaneously via thread pool (8 workers max). Writes run sequentially. Path-scoped tools targeting different files can also parallelize. clarify never runs in parallel.

Context Compression

Every LLM has a context window — max text it can process (e.g., 200K tokens). As conversations grow, the context fills. At 50% capacity, Hermes triggers compression: an auxiliary LLM summarizes older messages, preserving key info while freeing space. Last 20 messages always kept intact. Conversations can run indefinitely.

Iteration Budget

90 iterations per conversation (50 for subagents). Prevents infinite loops. Budget warnings injected as limit approaches.

Tools & Toolsets

A toolset is a logical grouping of related tools. Different platforms can enable different toolsets — for example, the CLI might enable browser tools while a Slack bot might not. This keeps the tool list focused and reduces token usage (every tool definition costs tokens in the prompt).

Memory System

Why memory matters: Without persistent memory, every conversation starts from zero. The agent wouldn't remember your name, your preferences, your project structure, or lessons learned from past mistakes. Memory transforms a stateless chatbot into a personalized assistant that improves over time.

MEMORY.md — Agent's Notebook

Stored at ~/.hermes/memories/MEMORY.md. This is the agent's personal notepad for facts about the environment, project conventions, tool quirks, and anything it wants to remember. Entries are delimited by § (section sign). Character limit: 2,200 chars (configurable). Example entries: "User's project uses Poetry for dependency management" or "The Docker container maps port 8080 to 3000."

USER.md — User Profile

Stored at ~/.hermes/memories/USER.md. Separate from MEMORY.md because user preferences and agent observations serve different purposes. USER.md stores: communication style preferences, technical skill level, workflow habits, timezone, name. Character limit: 1,375 chars. Keeping it separate lets the agent reason about "what I know about the world" vs. "what I know about this person."

Frozen Snapshot Pattern

Memory is injected into the system prompt once at session start. Mid-session writes update disk but NOT the running prompt — this preserves Anthropic's prompt cache (unchanged prompts reuse cached tokens, saving ~75%). Changes take effect next session.

Session Search (FTS5)

FTS5 (Full-Text Search 5) is SQLite's built-in search engine. It builds an inverted index — a mapping from every word to rows containing it — enabling fast keyword search. Hermes indexes every message from every session. The session_search tool searches past conversations by keyword, cheaper and faster than vector search for exact lookups.

Memory Nudge System

Every 10 user turns, the agent is nudged to update memory. After 6+ turns, a background LLM can extract facts automatically.

Skills System

A skill is a saved procedure — a Markdown document that teaches the agent how to perform a specific task. Think of it as a recipe card: "Here's how to deploy to Kubernetes" or "Here's how to create a GitHub PR." Skills are the mechanism by which the agent gets better over time.

How Skills Are Structured

skills/
└── software-development/
    └── deploy-to-k8s/
        ├── SKILL.md          # Main instructions (YAML frontmatter + markdown)
        ├── references/       # Supporting docs
        └── templates/        # Output templates

The SKILL.md file has YAML frontmatter (metadata at the top between --- markers) specifying: name, description, version, supported platforms, and prerequisites (required env vars, commands). The body contains step-by-step instructions in Markdown.

Progressive Disclosure (3 Tiers)

Loading all skill content into every prompt would waste tokens. Instead:

  • Tier 1 (skills_list): Only name + description are shown. Very cheap in tokens.
  • Tier 2 (skill_view): Full instructions loaded on demand when the agent decides a skill is relevant.
  • Tier 3 (skill_view + file_path): Reference files and templates loaded only when actively executing the skill.

Skills Hub

Integration with agentskills.io for discovering and installing community-created skills. The agent can search, install, and share skills from a public registry.

Self-Improvement Loop

After complex multi-tool tasks, the agent is nudged: "That was a complex workflow — would you like to save it as a skill?" A skills guard system scans new skills for safety (preventing malicious content). There are 26 built-in skill categories covering: software development, data science, research, creative writing, home automation, social media, and more — 70+ skills out of the box.

Prompt Caching & Cost Savings

The system prompt is identical every turn. Prompt caching marks it as cacheable — Anthropic stores processed tokens server-side, reusing them at ~25% cost. Hermes uses 4 cache breakpoints (system_and_3 strategy) for ~75% input cost reduction on multi-turn conversations.

Smart Routing & Fallback

Smart routing: Simple messages go to a cheap model (e.g., Gemini Flash); complex work uses the primary model. Fallback: On rate limit (429), overload (529), or service error (503), auto-switches to a fallback provider.

Platform Adapters

A platform adapter is a module that translates between a messaging platform's API and Hermes's internal message format. Each adapter handles: receiving messages (via webhooks, polling, or WebSockets), sending responses (text, media, files), managing platform-specific features (reactions, threads, stickers), and formatting output appropriately.

Gateway Architecture & Message Flow

The Gateway (hermes gateway start) is a single process managing all adapters simultaneously. Message flow: adapter receives message → checks DM auth → resolves session from SQLite → transcribes voice/caches images → creates AIAgent with platform toolsets → runs agent loop → splits response into platform chunks → delivers with media → updates session. The gateway also runs the cron scheduler (tick every 60s) and manages cross-platform channel directory.

Cron & Scheduling

What is cron? In Unix systems, cron is a time-based job scheduler — it runs tasks automatically at specified intervals (e.g., "every Monday at 9am" or "every 6 hours"). Hermes has its own cron system that lets the agent schedule recurring tasks.

How It Works

  • Jobs are stored in ~/.hermes/cron/jobs.json
  • The Gateway calls scheduler.tick() every 60 seconds from a background thread
  • The scheduler uses croniter (a Python library) to parse cron expressions and determine which jobs are due
  • For each due job: a fresh AIAgent is created, the job's prompt is executed (with optional skill loading), and output is saved
  • File-level locking prevents concurrent tick execution

Job Features

  • Natural language scheduling: You say "remind me every Monday at 9am to check server logs" and the agent creates the cron expression
  • Cross-platform delivery: Output can go to local (saved only), origin (the chat that created it), or a specific target like telegram:12345 or discord:67890
  • One-shot jobs: Run once at a specific time (with a 120-second grace period) then auto-delete
  • Silent mode: Jobs can output [SILENT] to save results without messaging the user
  • Management: Create, list, update, pause, resume, remove, or manually trigger jobs

Security

Command Approval System

Before executing potentially dangerous terminal commands, the approval system (tools/approval.py) checks against a policy. Commands can be auto-approved (safe), require user confirmation, or be blocked entirely. This prevents the agent from accidentally running rm -rf / or similar destructive operations.

Docker Isolation

The terminal backend can be configured to run inside a Docker container instead of the host machine. This provides OS-level isolation — even if the agent runs a malicious command, it's sandboxed inside a container with limited access to the host filesystem and network.

Tirith Security Engine

A policy engine (tools/tirith_security.py) that evaluates commands against configurable security rules. Named after the fortified city in Lord of the Rings, it provides defense-in-depth beyond simple command approval.

Additional Layers

  • PII/Secret Redaction: Automatically redacts API keys, passwords, and personal information from outputs (agent/redact.py)
  • Prompt Injection Scanning: Memory content is scanned for injection attempts before being stored
  • URL Safety: URLs are checked against blocklists before browser navigation
  • Website Blocklist: Configurable list of domains the browser cannot visit
  • Skills Guard: New skills are scanned for malicious content before being saved
  • Iteration Budgets: Hard limits prevent runaway tool loops (90 for main agent, 50 for subagents)
